## What Was NOT Implemented (Deferred)

### Progress Spinners

**Reason:** Would require significant refactoring of cleaner interface to support async progress callbacks. Current architecture doesn't have a way to report progress during long operations like Nix garbage collection.

**Alternative considered:** Could add `--verbose` flag to show spinner during operations, but this would be a larger change to the command flow.

**Decision:** Deferred to future enhancement when cleaner interface is refactored for async support.

## Type Model Architecture

**Pattern used:** Interface segregation

```go
// Base interface (all cleaners implement)
type Cleaner interface {
    Name() string
    Type() domain.OperationType
    Clean(ctx context.Context) result.Result[domain.CleanResult]
    IsAvailable(ctx context.Context) bool
    Scan(ctx context.Context) result.Result[[]domain.ScanItem]
}

// Optional interface (Nix cleaner implements)
type NixStoreSizer interface {
    GetStoreSize(ctx context.Context) int64
}
```

This allows the scan command to optionally query store size without requiring all cleaners to implement it.
